# Currying

## Currying이란

수학 방정식에서 유래

```javascript
const add = (x, y) => x + y;
```

다음 함수를 수학적으로 표시하면 다음과 같다.

```javascript
f(x, y) = x + y
```

이 표기법은 다음과 같다.

```javascript
f(x, y) = h(x)(y) = x + y
```

`h(x)(y)`는 값을 하나 받아 다른 값을 받는 함수를 반환하는 형태의 함수이다.  
(왜 h(x)(y)가 되는지 모르겠음. 함수 표기법을 모르겠다.)  
이 때 `x = 2`를 만족한다면 식은 다음과 같다.

```javascript
f(2, y) = h(2)(y) = 2 + y
```

위의 `add()`를 위와 같은 표현식으로 바꿔보자.

```javascript
// Arrow function 
const add = x => y => x + y;
}
```

자바스크립트는 함수가 일급 객체로 취급되기 때문에 함수가 함수를 반환할 수 있다. 이를 [[Higher Order Function]]이라 한다.

이렇게 표현된 add는 다음과 같이 사용할 수 있다.

```javascript
const addFive = add(5);
addFive(7); // 12
```

즉 `Currying`이란 1개의 파라미터를 받는 함수를 리턴하는 함수를 리턴 -> 리턴 -> 리턴하는 함수이다. 매 번마다 받을 수 있는 인자는 오직 하나이다.
