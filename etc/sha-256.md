# Sha 256

SHA256은 결과물이 256bit, 32byte이며 Base64는 3byte를 4글자로 (6bit per character) 치환합니다.
따라서 32byte / 3byte = 10.6666 이기 때문에 글자로는 40글자가 나오고, 나머지 바이트는 연산 후 padding character '='를 포함해 4자가 더 붙기 때문에 총 44자가 되어야 합니다.

MySQL Function sha2('test1',256)은 레퍼런스에 따르면 Non-binary string을 반환합니다. (As of MySQL 5.5.6, the return value is a nonbinary string in the connection character set.)
256bit를 HEX로 치환한 string을 반환하기 때문에 4bit씩 잘라 HEX string으로 변환해 256 / 4 = 64자 string으로 결과가 나오게 됩니다. (2^4 = 16이므로)
1character = 8bit이므로 64 * 8 = 512bit, Base64 펑션에는 HEX string 512비트가 들어가게 됩니다.

올바른 해시값을 얻으려면 SELECT TO_BASE64(UNHEX(sha2('test1',256)))로 HEX string을 bit로 변환해서 사용하셔야 합니다.
