# Celery Beat 사용 시 문제점

- 장고 특성 상 [[celery-beat]] 에서도 로직을 수행하려면 모델 클래스가 필요함
  - 모델 공유를 어떻게 할 것인가에 대한 문제가 발생
- 모델 공유 방안에 대해 리서치 진행 
  - [[share-model]]
- 결론적으로 동일 프로젝트에 구성하고, 인스턴스를 다르게 배포하기로 결정
  - 기존 모델을 그대로 사용 가능
  - 기존 로직을 그대로 사용 가능
  - 라이브러리, `celery.py`, task 파일 세가지로 코드 변경이 많지 않음
- 궁극적으로는 MSA 구성을 하는것이 맞을 듯
